Recommended check

Complete these steps in order. Most missing banners are caused by the operating system's per-app settings, not by the extension.

  1. Send a test. Use the button above and wait at least five seconds.
  2. Open the notification center. The alert may be stored there even when no pop-up banner appeared.
  3. Allow Google Chrome in the operating system. Enable banners, notification center/history, and sounds.
  4. Turn off Do Not Disturb or Focus. Also inspect automatic schedules and gaming/full-screen rules.
  5. Keep Chrome running. Island Watcher cannot check maps or notify while every Chrome process is closed.
  6. Retest. After changing settings, fully restart Chrome and send another test.
Important Do not uninstall the extension while troubleshooting. Uninstalling can remove locally saved favorites. Use Reload on the extensions page instead.

Google Chrome checks

These checks apply on Windows, macOS, and Linux.

1. Confirm that Island Watcher is enabled
  1. Open chrome://extensions/.
  2. Find Fortnite.GG Island Watcher.
  3. Confirm that its toggle is enabled.
  4. Click Reload after installing an updated folder.
  5. If an Errors button appears, open it and review the latest error.
2. Check Chrome's notification settings
  1. Open Chrome Settings.
  2. Go to Privacy and security → Site settings → Notifications.
  3. Make sure Chrome is not configured to block all notification requests.
  4. This page primarily controls websites. Extension alerts use the manifest's notifications permission, but this check helps confirm that Chrome notifications are not broadly restricted.
3. Allow Chrome to continue in the background
  1. Open chrome://settings/system.
  2. If your Chrome version shows Continue running background apps when Google Chrome is closed, enable it.
  3. Even with this option enabled, restarting or shutting down the computer pauses Island Watcher until Chrome runs again.
4. Check for managed-browser restrictions
  1. Open chrome://policy.
  2. Look for notification, extension, or background-app policies.
  3. If Chrome says Managed by your organization, a school or workplace administrator may control notifications.

Windows notification checks

Google Chrome must be allowed as an app sender. The main notifications switch can be on while Chrome's banners remain off.

Windows 11 — complete checklist
  1. Press Win + I to open Settings.
  2. Go to System → Notifications.
  3. Turn the main Notifications switch on.
  4. Under Notifications from apps and other senders, find Google Chrome and turn it on.
  5. Click Google Chrome, then enable Show notification banners, Show notifications in notification center, and Play a sound when a notification arrives.
  6. Return to the Notifications page and turn Do not disturb off.
  7. Expand Turn on do not disturb automatically and check scheduled, full-screen, gaming, and display-duplication rules.
  8. Press Win + N to inspect Notification Center.
ms-settings:notifications
Press Win + R, paste the command, and press Enter.
Windows 10 — complete checklist
  1. Open Start → Settings → System → Notifications & actions.
  2. Enable Get notifications from apps and other senders.
  3. Under sender settings, enable Google Chrome.
  4. Click Google Chrome and enable banners, Action Center, and sounds.
  5. Open Action Center and set Focus assist to Off.
  6. Check Focus assist automatic rules, especially gaming and full-screen rules.
Sound and background checks
  1. Open Settings → System → Sound → Volume mixer.
  2. Confirm that Google Chrome is not muted and its app volume is above zero.
  3. Check the taskbar system tray to confirm Chrome still has a running process when all browser windows are closed.
  4. After changing Windows notification settings, fully exit Chrome, reopen it, and run the test again.
If Google Chrome is missing from the sender list
  1. Keep the Windows Notifications page open.
  2. Return to Island Watcher and send a test notification.
  3. Reopen or refresh the Windows Notifications page.
  4. Restart Chrome, then restart Windows if Chrome still does not appear.

macOS notification checks

Chrome must be allowed in macOS Application Notifications, and Focus must allow the alert.

macOS Ventura and newer
  1. Choose Apple menu → System Settings.
  2. Click Notifications in the sidebar.
  3. Under Application Notifications, select Google Chrome.
  4. Turn Allow notifications on.
  5. Select Banners or Alerts as the notification style.
  6. Enable Show in Notification Center and Play sound for notification.
  7. Choose Apple menu → System Settings → Focus. Turn Focus off or allow Google Chrome in the active Focus.
  8. Click the date and time in the menu bar to inspect Notification Center.
Older macOS versions
  1. Open Apple menu → System Preferences → Notifications & Focus.
  2. Select Google Chrome.
  3. Enable notifications, choose Banners or Alerts, and enable sounds.
  4. Open the Focus tab and disable Do Not Disturb or permit Chrome.
Additional macOS checks
  1. Make sure Chrome is running. Island Watcher cannot execute after Chrome has fully quit.
  2. Check whether Screen Sharing, gaming, or another Focus automatically enabled Do Not Disturb.
  3. Restart Chrome after changing notification permission.
  4. If Chrome does not appear in Application Notifications, send a test first and reopen System Settings.

Linux notification checks

Menus vary by desktop environment. Start with the guide matching GNOME/Ubuntu or KDE Plasma.

GNOME and Ubuntu
  1. Open Activities, search for Settings, and open it.
  2. Select Notifications in the sidebar.
  3. Turn notifications on globally.
  4. Find Google Chrome or Chromium and enable its notifications.
  5. Turn Do Not Disturb off from the system menu or notification panel.
  6. Click the clock/date area—or press Super + V on supported GNOME versions—to inspect notification history.
  7. Open Settings → Sound and verify alert volume and alert sound.
KDE Plasma
  1. Open System Settings → Notifications.
  2. Open application-specific settings and select Google Chrome or Chromium when available.
  3. Enable pop-up notifications, notification history, and sound.
  4. Open the notification icon in the system tray and turn Do Not Disturb off.
  5. Run another Island Watcher test.
Test the Linux notification service
  1. Open a terminal.
  2. Run the command below if notify-send is installed.
  3. If this system test also fails, troubleshoot the desktop notification daemon before Chrome.
notify-send "Notification test" "Linux notification service is working"
Sandboxed Chrome or Chromium packages
  1. If the browser was installed through Snap, Flatpak, or another sandbox, check its app permissions in your software center or sandbox permission manager.
  2. Confirm that desktop integration and notification portals are allowed.
  3. Restart the browser after changing sandbox permissions.

Advanced troubleshooting

Use these steps after both Chrome and operating-system settings look correct.

Check the extension service worker
  1. Open chrome://extensions/.
  2. Enable Developer mode.
  3. Find Island Watcher and click the service worker link.
  4. Keep the console open, send a test notification, and look for red errors.
  5. Use Reload instead of uninstalling the extension.
Confirm the correct Chrome profile
  1. Check the profile icon in the top-right corner of Chrome.
  2. Confirm Island Watcher is installed in the same profile currently running.
  3. Operating-system notification settings apply to Google Chrome as an app, while extension installation is profile-specific.
Restart sequence
  1. Save your work.
  2. Close every Chrome window.
  3. Use Task Manager, Activity Monitor, or your Linux system monitor to confirm Chrome has stopped.
  4. Open Chrome again, reload the extension, and send a test.
  5. If it still fails, restart the operating system and repeat the test.
What each result means
  • Permission: granted, but no banner: the operating system or desktop environment is likely suppressing Chrome.
  • Notification appears only in history: banners are disabled while notification-center/history delivery is enabled.
  • Banner appears without sound: app sounds, alert volume, or system sound settings are disabled.
  • Tests work only while Chrome is open: background Chrome processes are not remaining active.
  • Nothing works, including an OS-level test: troubleshoot the operating-system notification service first.
Island Watcher does not detect or change operating-system notification settings. This page provides checks and opens the relevant Chrome pages.
